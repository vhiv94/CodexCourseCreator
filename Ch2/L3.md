# L3: Choose What Happens Next

## Concept

So far, every typed line becomes “the answer.” Real tools need **guards**: rules that say when an answer is too thin to use. In Python, **`if` / `else`** chooses between two (or more) blocks: one runs when a **condition** is true, another when it is not. The condition often checks the **stripped** text you already learned about—spaces and tabs around the edges do not count once you `.strip()`.

When the stripped line is **empty**, there is nothing meaningful to treat as a study note. The program should **not** pretend the note was fine. It should **tell the user plainly** what went wrong—still on **stdout**, so the tests (and a human) can see the message—then finish without crashing.

```python
# Illustration only: branching after strip (not a full solution).
clean = raw.strip()
if not clean:
    print("That note was empty after trimming spaces.")
else:
    print(clean)
```

## Why it matters in this project

The study log will only be trustworthy if empty noise never masquerades as a real entry. This lesson adds the smallest useful gate—**one line, strip, then decide**—before later lessons add loops, files, and richer validation.

## Common misconceptions

- **“I can just print nothing when it’s empty.”** — The tests expect a **visible** rejection message on stdout, not silence.
- **“Two separate `if` blocks are the same as if/else.”** — This lesson’s source check looks for **`if` with a non-empty `orelse`** (an `else` or `elif` chain). Match that shape so the static test can confirm you modeled both paths as one decision.
