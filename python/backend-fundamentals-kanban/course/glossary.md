# Glossary

## API
Application Programming Interface. In this course, the HTTP contract clients use to create and manage boards, columns, and tasks.

## Background Job
Work processed outside the main request path (for example activity fan-out) so user-facing requests stay fast.

## Cache
Temporary data storage used to speed up repeated reads, usually with explicit invalidation rules.

## CI/CD
Continuous Integration and Continuous Delivery/Deployment. Automated checks and release steps that reduce shipping risk.

## Container
An isolated runtime unit that packages application code and dependencies with predictable behavior across environments.

## DNS
Domain Name System. The lookup system that maps domain names to IP addresses.

## Foreign Key
A relational constraint linking a row in one table to a row in another table.

## Health Check
An endpoint or probe that reports whether a service is alive and ready to serve traffic.

## HTTP Status Code
Numeric response code indicating result class, such as success (2xx), client error (4xx), or server error (5xx).

## Idempotent
An operation that can be repeated with the same outcome and no additional side effects (for example many PUT operations).

## Index
A data structure maintained by the database to accelerate lookups and sorting at additional write/storage cost.

## Migration
A versioned schema change that can be applied in order to move a database from one structural state to another.

## Observability
The ability to understand system behavior from logs, metrics, traces, and health signals.

## Reverse Proxy
An edge service that receives incoming traffic and forwards it to upstream app services.

## SQLite
A lightweight relational database engine stored in a local file, useful for embedded and self-hosted workloads.

## TTL
Time To Live. DNS cache lifetime that controls how long resolvers keep a record before refreshing.

## VPS
Virtual Private Server. A remote host where you deploy and run your self-hosted backend services.
