# Build a Replenishment Queue Class

Where stacks optimize undo semantics, queues optimize fairness and arrival order. This lesson defines a FIFO queue class for replenishment jobs so earlier requests are handled before later ones unless explicit prioritization is introduced in future chapters. The key tools are class-based encapsulation, queue operations (`enqueue`, `dequeue`, `peek`), and explicit empty-queue error behavior.

This matters because operational systems become unpredictable when work ordering drifts unintentionally. FIFO is a policy choice that keeps behavior understandable for dispatch and planning teams. A common misconception is that dequeueing from an empty queue should return `None`; this often hides workflow bugs. Another is treating `peek` as optional, when non-destructive inspection is important for orchestration code. Reflection prompt: which production bug would be hardest to trace if empty queue reads silently returned placeholder values?
