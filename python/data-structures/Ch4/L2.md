# Use Sets to Enforce Location and SKU Constraints

Sets give fast, explicit membership boundaries for operational policy checks. This lesson applies set-based filtering to eliminate candidates that violate blocked locations or restricted SKU rules before planning proceeds. The primary tools are set membership operations, predictable filter passes, and clear policy precedence when multiple constraints apply.

This is critical because policy violations are often expensive and discovered too late if not enforced early. The tradeoff is strictness versus flexibility: over-filtering can reduce fulfillment options, while under-filtering can create invalid picks. Common misconception: constraints are a UI concern only. In robust systems, they are core data-plane rules. Reflection prompt: if a candidate fails both location and SKU policy checks, does your pipeline need to track both reasons or just exclusion outcome?
