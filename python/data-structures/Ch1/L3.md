# Index Current Stock by SKU and Location

After event normalization, you need a structure that answers operational questions quickly: \"what do we have, and where?\" This lesson introduces a nested dictionary index keyed first by SKU, then by location, with quantities as values. The practical tools are controlled dictionary initialization, additive updates, and invariant checks such as non-negative stock where that rule is expected by your design.

This matters because every later allocation and replenishment decision depends on this index being trustworthy and cheap to query. The tradeoff is between a simple but rigid shape and a flexible but inconsistent one; the tests intentionally reward consistency. Common misconceptions include treating location as optional once SKU totals are known and overwriting quantity instead of aggregating movement effects. Reflection prompt: how does preserving location-level detail prevent bad allocation decisions later?
