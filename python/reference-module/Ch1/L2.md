# Append a Clean Note

## Concept

This lesson extends deterministic string handling into state updates. `append_note(notes: list[str], raw_note: str) -> list[str]` should normalize input with `str.strip()`, validate that the cleaned note is not empty, and raise `ValueError` when the contract is violated. A list-update pattern then returns the new note state in order.

The project reason for this step is data integrity: once notes enter state, bad input propagates into every later feature. A key tradeoff is mutation versus returning a new list, where returning a new value often reduces side effects and improves test clarity. Common misconceptions include validating before normalization (which misses whitespace-only inputs) and assuming append behavior must mutate in place to be correct.

## Assignment

Implement `append_note(notes: list[str], raw_note: str) -> list[str]` in `main.py` so it trims input, rejects empty normalized notes with `ValueError`, and appends valid cleaned notes in order.

Expected outcomes:
- `append_note(["first"], " second  ")` returns `["first", "second"]`.
- `append_note(["first"], "   ")` raises `ValueError`.
- Existing entries remain in order and the new cleaned note is appended at the end.

**Evidence (tests):** `main_test.py` + `lesson_ch1_l2`  
Run with: `uv run pytest main_test.py -m lesson_ch1_l2`
