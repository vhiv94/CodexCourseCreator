# Format a Greeting

## Concept

This lesson introduces a small pure-function contract: `format_greeting(name: str) -> str` should return deterministic output from input text only. The tools in play are `str.strip()` to normalize user-provided edge whitespace, f-strings to produce exact punctuation and spacing, and selector-scoped pytest execution to confirm behavior in isolation.

It matters to this module's arc because the reference module is meant to model predictable, testable lesson increments. If greeting formatting is inconsistent now, follow-up state and validation lessons are harder to reason about. A common misconception is that "readable enough" output is acceptable; this contract is exact. Another misconception is that `strip()` removes interior spacing, but it only trims the edges.

## Assignment

Implement `format_greeting(name: str) -> str` in `main.py` so it trims surrounding whitespace and returns the exact greeting shape `Hello, <name>!`.

Expected outcomes:
- `format_greeting("  Ada  ")` returns `Hello, Ada!`.
- `format_greeting("Sam")` returns `Hello, Sam!`.
- Repeated calls with the same input return identical output.

**Evidence (tests):** `main_test.py` + `lesson_ch1_l1`  
Run with: `uv run pytest main_test.py -m lesson_ch1_l1`
