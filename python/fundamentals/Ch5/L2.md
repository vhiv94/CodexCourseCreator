# Use the Main Guard Correctly

The main guard separates "code that defines behavior" from "code that runs behavior." In this lesson you ensure `main()` exists as an explicit orchestration point and can be called under `if __name__ == "__main__":`. The tools are function definition and Python's module execution model. This project step keeps imports safe and prevents accidental side effects when tests load the module. A common misconception is that main guard is optional style; for script-like modules, it is a practical boundary.

## Assigned task(s)

- Define a callable `main()` function in `main.py`.
- Ensure module structure supports direct execution and import-based testing.
- Keep top-level side effects minimal.
