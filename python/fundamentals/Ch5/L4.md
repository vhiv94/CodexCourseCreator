# Dispatch Basic Commands

Command dispatch maps user intent to one action at a time. In this lesson you implement a small routing decision for `add`, `list`, and `summary` commands. The tools are string normalization and conditional branching. This project step is the first true interaction layer for your tracker and prepares later file persistence and object integration. A common misconception is that command parsing must be complex from day one; simple explicit branching is often the best first version.

## Assigned task(s)

- Create `dispatch_command(command)` in `main.py`.
- Return normalized command identifiers for supported commands.
- Keep command handling predictable for `add`, `list`, and `summary`.
- **Evidence (tests):** `main_test.py` + `lesson_ch5_l4`
