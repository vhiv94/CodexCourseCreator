# Organize Code with Modules and Imports

Modules let you split code by responsibility, and imports let those pieces collaborate. In this lesson you formalize a small public API so tests and future files know which names are intentional entry points. The tools are module-level constants and import-safe organization. This project step makes your tracker easier to navigate as it grows beyond one tiny script. A common misconception is that "one file is always simpler"; simplicity changes as project size grows.

## Assigned task(s)

- Define a `PUBLIC_API` tuple in `main.py`.
- Include key callable names that represent your intended external surface.
- Keep the API list stable and explicit.
- **Evidence (tests):** `main_test.py` + `lesson_ch5_l1`
