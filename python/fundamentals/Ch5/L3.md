# Sequence Work in main

Orchestration is about order: load state, decide action, update state, then render output. In this lesson you describe that execution plan explicitly so the project flow stays understandable as complexity grows. The core tool is a step plan representation that can be tested before full command handling exists. This project step ties together earlier helpers into a coherent runtime sequence. A common misconception is that "Python finds the right order automatically"; the program follows the sequence you define.

## Assigned task(s)

- Create `plan_main_steps()` in `main.py`.
- Return the planned command flow in a clear ordered sequence.
- Keep names readable so each stage communicates intent.
- **Evidence (tests):** `main_test.py` + `lesson_ch5_l3`

Optional implementation sketches live in `Ch5/L3-hint.md`.
