# Use Conditionals for Decisions

Conditionals let your program choose behavior based on rules. In this lesson you define what counts as a valid entry and return a boolean decision. The tool focus is `if` logic tied to normalized input, because explicit conditions are easier to reason about than hidden assumptions. This project step protects tracker state from invalid data before lists and files are updated. A common misconception is that every invalid case should raise an exception; sometimes a simple `True`/`False` contract is the better first boundary.

## Assigned task(s)

- Create `is_valid_entry(text)` in `main.py`.
- Return `False` for invalid input and `True` for valid input.
- Base the decision on clear, readable conditionals.
- **Evidence (tests):** `main_test.py` + `lesson_ch2_l4`
