# Run Your First Program

A Python script runs in sequence from top to bottom, and your first goal is to make that execution visible with one small message. In this lesson you use a function boundary as a tool for clarity: instead of printing everywhere, return a welcome string from one place so tests can check it directly. This project step starts the Personal Tracker with a stable opening message and teaches an important beginner habit: separate "compute a value" from "display a value." A common misconception is that code order does not matter, but in Python declaration and call order is essential.

## Assigned task(s)

- Create `build_welcome_message()` in `main.py` and make it return a string.
- Ensure the returned text clearly includes "Personal Tracker" so intent is obvious.
- Keep the behavior deterministic so repeated calls return the same message.
- **Evidence (tests):** `main_test.py` + `lesson_ch1_l1`
