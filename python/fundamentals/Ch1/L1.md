# L1: Hello from Python

## Concept

**Python** is a language people use to give precise instructions to a computer. You usually store those instructions in a **plain text file** (often ending in `.py`). A program called the **Python interpreter** reads the file and follows the instructions from top to bottom.

When the interpreter runs your file, it can **print** characters to the **standard output** stream—what you typically see in the terminal as normal program output. In Python, the built-in function **`print(...)`** is the usual way to show a line of text there. Each call can add a line break at the end unless you change that behavior (advanced topic for later).

A short script can do almost nothing except print one message. If the interpreter finishes without hitting an error, the process **exits successfully**, which other tools (including tests) often record as **exit status 0**. That is a simple signal meaning “this run completed as expected.”

This lesson keeps the program tiny on purpose: one file, one clear message—so you practice **running** Python and **seeing** output, which every later lesson will build on.

## Why it matters in this project

The course grows one **command-line study log**. Before you can add notes, files, or menus, you need to trust that you can run your program and read what it prints. This lesson establishes that rhythm: edit `main.py`, run it from the terminal, confirm the output. The welcome line you print is the first visible tie between “my code” and “what the user sees”—the same channel the finished log will use for prompts and listings.

## Common misconceptions

- **“I printed twice and it still looks like one message.”** — Two `print` calls usually mean **two lines**. The tests count lines in the captured output; one visible line means one line break pattern in the output, not two separate prints (unless you deliberately print on one line without a newline in between—keep it simple: one line, one clear approach).

- **“Empty `print()` is fine.”** — Printing nothing useful can produce a blank line or empty output. The tests require **visible text** on that single line.

- **“The file can live anywhere.”** — For this course, the tests look for **`python/fundamentals/src/main.py`**. Putting the script somewhere else will not satisfy the fixture that opens your entrypoint.
