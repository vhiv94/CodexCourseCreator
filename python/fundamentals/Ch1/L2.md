# Name Values with Variables

Variables give names to values so your code can explain itself. In this lesson you will use assignment and dictionary construction to store app metadata such as app name, version, and debug mode. The tool choice is intentional: a dictionary is easy to inspect and keeps related values grouped while you are still learning. This project step turns your tracker from a single message into a tiny configured application. A common misconception is that variable names are just decoration; in practice, names are part of program design and communication.

## Assigned task(s)

- Create `build_app_metadata()` in `main.py`.
- Return a dictionary with at least `app_name`, `version`, and `debug_mode`.
- Use clear Pythonic names (`snake_case`) for both variables and keys.
- **Evidence (tests):** `main_test.py` + `lesson_ch1_l2`
