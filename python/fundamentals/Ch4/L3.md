# Understand Local Scope

Scope answers "where is this name visible?" and protects functions from accidental cross-talk. You will write a small helper that uses only passed-in values so behavior does not depend on hidden globals. The tools are parameters, local variables, and explicit returns. This project step makes later refactors safer because each function has clear boundaries. A common misconception is that global constants are always bad; the real issue is uncontrolled mutable global state.

## Assigned task(s)

- Create `increment_minutes(current, delta)` in `main.py`.
- Compute and return the new total from local values only.
- Keep behavior consistent regardless of external module variables.
- **Evidence (tests):** `main_test.py` + `lesson_ch4_l3`
