# Define Functions with Parameters

Functions package repeatable logic behind a clear interface. In this lesson you practice function signatures with named parameters so call sites read like small contracts. The tools are `def`, parameters, and return values rather than side effects. This project step standardizes how tracker entry formatting is reused across the program. A common misconception is that long parameter lists always mean bad design; sometimes explicitness is better while learning.

## Assigned task(s)

- Create `format_entry_line(name, category, minutes)` in `main.py`.
- Return one formatted string using the three inputs.
- Keep output formatting stable and readable.
- **Evidence (tests):** `main_test.py` + `lesson_ch4_l1`
