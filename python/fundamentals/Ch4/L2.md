# Return Values from Functions

Returning values makes behavior testable and composable. In this lesson you summarize tracker entries by returning a dictionary instead of printing directly. The tool choice encourages separation of concerns: computation first, presentation later. This project step enables other functions to reuse summary data without re-parsing output text. A common misconception is that printing is equivalent to returning; printed text cannot be easily reused as data.

## Assigned task(s)

- Create `build_summary(entries)` in `main.py`.
- Return a dictionary with count and total minutes.
- Avoid relying on print-only side effects for correctness.
- **Evidence (tests):** `main_test.py` + `lesson_ch4_l2`
