# Lock Records with Tuples

Tuples are ordered like lists but immutable, which makes them useful for fixed-shape records. You will represent one tracker entry as a tuple so each position has a stable meaning. The main tool is structural consistency: the same order every time prevents downstream confusion. This project step prepares data for dictionary conversion and class modeling in later chapters. A common misconception is that tuples are "better lists"; they solve a different problem: fixed structure.

## Assigned task(s)

- Create `make_entry_tuple(name, category, minutes)` in `main.py`.
- Return a tuple with a stable field order.
- Keep the tuple shape consistent across calls.
