# Store Sequences in Lists

Lists are ordered and mutable, which makes them a practical first container for tracker state. In this lesson you model "all entries so far" as a list and return an updated list when a new entry is added. The tool choice reinforces an invariant: adding one item should not accidentally erase earlier items. This project step introduces stable in-memory state management before persistence. A common misconception is that mutating input is always harmless; hidden mutation can surprise callers.

## Assigned task(s)

- Create `add_entry(entries, new_entry)` in `main.py`.
- Return a list with `new_entry` added in order.
- Keep earlier entries intact and readable.
- **Evidence (tests):** `main_test.py` + `lesson_ch3_l1`
