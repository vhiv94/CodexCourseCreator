# Handle Exceptions Gracefully

Exceptions represent runtime failures, and graceful handling keeps the user experience stable. In this lesson you safely parse a value that may be invalid and return a fallback when parsing fails. The main tools are `try/except` and clear default values. This project step makes command handling resilient when users type unexpected data. A common misconception is that catching all exceptions is always safe; targeted exception handling is usually better.

## Assigned task(s)

- Create `safe_parse_minutes(raw_value, default=0)` in `main.py`.
- Return a parsed integer when possible, otherwise return the default value.
- Keep error handling readable and bounded to expected failure cases.
