# Initialize Objects with __init__

`__init__` sets up object state when an instance is created. In this lesson you define required fields so each `TrackerItem` has predictable attributes from the start. The tool is constructor design: clear required inputs make downstream behavior easier to reason about. This project step gives your tracker objects stable structure. A common misconception is that `__init__` creates the object itself; object creation happens before initialization.

## Assigned task(s)

- Preserve values exactly so later methods can rely on them.
