# Add Behavior with Instance Methods

Instance methods turn stored object state into computed behavior. In this lesson you add a `status_line()` method that transforms one `TrackerItem` into user-facing output. The tool focus is `self`: methods read and combine attributes from the current instance. This project step moves logic closer to the data it operates on, which simplifies orchestration code. A common misconception is that every helper should be a method; methods are best when behavior is tightly coupled to instance state.

## Assigned task(s)

- Implement `TrackerItem.status_line(self)` in `main.py`.
- Return a string built from the instance's stored values.
- Keep output deterministic for repeated calls on the same object.
- **Evidence (tests):** `main_test.py` + `lesson_ch7_l4`
