# Compare __str__ and __repr__

Objects can present two different string views: one for end users and one for developers. In this lesson you define both `__str__` and `__repr__` on `TrackerItem` and compare their intent. The tools are special methods and audience-aware representation. This project step improves both CLI readability and debugging quality. A common misconception is that these methods should always return identical text; they often serve different communication goals.

## Assigned task(s)

- Implement both `__str__` and `__repr__` on `TrackerItem`.
- Ensure each returns non-empty text with distinct purpose.
- Keep representations consistent with tracker domain language.
