# Use Polymorphism for Summaries

Polymorphism means you can call one method name across different object types and still get type-specific behavior. In this lesson you build a summary renderer that trusts a shared object interface instead of checking each concrete type manually. The tool is method dispatch through a common base contract. This project step keeps reporting flexible as new tracker item types are added. A common misconception is that polymorphism requires complex frameworks; basic method overriding already demonstrates it.

## Assigned task(s)

- Create `render_item_summaries(items)` in `main.py`.
- Call each item's shared summary interface and collect the results.
- Return a list of summary strings.
- **Evidence (tests):** `main_test.py` + `lesson_ch8_l2`
