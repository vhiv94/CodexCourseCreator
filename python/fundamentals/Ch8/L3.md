# Integrate Functional and OOP Layers

Real programs mix styles: functions for pipelines and classes for domain objects. In this lesson you connect tuple-based data flow with object-based reporting. The tools are conversion boundaries and disciplined layering between helpers and models. This project step ensures earlier function work remains useful rather than discarded. A common misconception is that switching to classes means rewriting everything; integration usually works better than replacement.

## Assigned task(s)

- Create `build_tracker_report(entries)` in `main.py`.
- Convert entry data into object-level summaries through your designed interfaces.
- Return one combined report string for display.
- **Evidence (tests):** `main_test.py` + `lesson_ch8_l3`
