# Build a Base Class and Child Classes

Inheritance lets related classes share behavior while specializing details. In this lesson you create a `BaseTrackerItem` and at least two child classes for specific tracker domains. The tools are parent-child class definitions and `issubclass` relationships. This project step introduces extensibility without rewriting common fields repeatedly. A common misconception is that inheritance is always better than composition; here it is useful because shared structure is explicit and small.

## Assigned task(s)

- Define `BaseTrackerItem`, `FitnessTrackerItem`, and `LearningTrackerItem` in `main.py`.
- Ensure child classes inherit from the base class.
- Keep shared state and behavior centralized in the parent where appropriate.
- **Evidence (tests):** `main_test.py` + `lesson_ch8_l1`

Optional implementation sketches live in `Ch8/L1-hint.md`.
