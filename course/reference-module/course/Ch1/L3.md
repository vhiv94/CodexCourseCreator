# Render the Study Summary

Composing two tiny helpers into one stable output is easier when each earlier lesson already settled its own rules. This lesson uses string assembly, newline joins, and a simple fallback bullet to keep the final summary deterministic even when the note list is empty. It matters because this is the first time the project exposes a complete learner-facing artifact instead of isolated helper behavior. A common mistake is to let the formatting drift between cases, which makes the summary harder to scan and harder to verify.

## Assigned task(s)

- **Build/Change:** Implement `build_progress_summary(raw_name, raw_notes)` and update `main()` to print one example summary.
- **Expected outcomes / acceptance criteria:**
  - [ ] The final summary uses the cleaned name from `normalize_name(...)`.
  - [ ] The final summary uses the cleaned notes from `collect_notes(...)`.
  - [ ] The output stays multi-line and deterministic, with `- No practice notes yet.` when no valid notes remain.
- **Evidence:** Show the printed summary. An optional targeted test run is also valid supporting evidence.
- **Optional test evidence:** `main_test.py` + `lesson_ch1_l3`
