# Normalize a Learner Name

Small helper functions are useful when they turn messy input into one predictable shape that later code can trust. In this lesson, the important tools are `str.strip()` for trimming edge whitespace and a simple fallback value for the blank-input case. This matters to the project because every later output line should use the same cleaned name instead of re-handling raw input in multiple places. A common mistake is to assume "someone will probably type a real name," but tiny scripts become easier to reason about when even awkward input has a clear rule.

## Assigned task(s)

- **Build/Change:** Implement `normalize_name(raw_name)` in `main.py`.
- **Expected outcomes / acceptance criteria:**
  - [ ] Surrounding whitespace is removed before the name is used anywhere else.
  - [ ] Blank or whitespace-only input falls back to `Learner`.
  - [ ] The function stays focused on one transformation and does not print or handle notes.
- **Evidence:** A short demo call or brief explanation of the cleanup rule is enough for this lesson.
