# Keep Only Real Practice Notes

Once a project starts carrying user notes forward, input cleanup becomes a data-quality decision rather than a cosmetic one. Here the useful tools are list iteration, `str.strip()`, and a clear rule for skipping entries that become empty after cleanup. This lesson advances the project by giving the summary builder a stable list of meaningful notes instead of raw noisy lines. A common misconception is that empty strings are harmless, but they usually become awkward blank bullets and force later code to defend against avoidable noise.

## Assigned task(s)

- **Build/Change:** Implement `collect_notes(raw_notes)` in `main.py`.
- **Expected outcomes / acceptance criteria:**
  - [ ] Each raw note is stripped before the script decides whether to keep it.
  - [ ] Notes that are empty after cleanup are skipped.
  - [ ] Valid notes keep their original order in the returned list.
- **Evidence:** Show one before/after example or briefly explain the filter rule you chose.
