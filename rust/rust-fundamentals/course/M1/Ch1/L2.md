# Read the crate layout and `main`

**Cited contract:** [`course/spine.yaml`](../../spine.yaml) (M1 → Ch1 → L2), [`course/overview.md`](../../overview.md) (Log Summary CLI arc). **Builds on:** [L1 — Create and run the Log Summary crate](L1.md).

A binary crate is mostly **a manifest plus a tree of Rust source**. Cargo reads the manifest to learn the package name, edition, and (later) dependencies and target configuration; the compiler then turns your `src/` files into one runnable program. That split is deliberate: humans and tools edit **text** in `Cargo.toml` and `.rs` files, while `target/` holds generated binaries and cache—treat `target/` as disposable output, not something you hand-edit.

For any Rust program you run, the **first code path** you need a mental map of is **`fn main()`** in the binary’s root source file. The operating system starts a process; the Rust runtime hands off to your `main`, and everything else you write is either called from there or from functions `main` eventually calls. If you are used to a thin `main` that delegates to a “loop” or app module, Rust supports that pattern well—you will wire it in once modules are on the table. For now, stay inside `src/main.rs` and focus on **where execution begins** and **where metadata lives**.

`Cargo.toml` uses TOML: tables like `[package]` hold identity fields (name, version, edition). You do not need every key memorized this lesson; you need to recognize the file as **the crate’s configuration root** and know where to look when Cargo or a tutorial mentions the manifest.

## Assigned task(s)

- **Build/Change:** Open your existing **Log Summary CLI** crate from L1. Read **`Cargo.toml`** and **`src/main.rs`** end-to-end without restructuring the project yet.
- **Expected outcomes / acceptance criteria:**
  - [ ] You can point to **`Cargo.toml`** as the place **package metadata** (at least name and Rust **edition**) is declared for this crate.
  - [ ] You can point to **`fn main()`** in **`src/main.rs`** and explain, in your own words, why that function is the **first Rust code path** that runs when you execute the binary.
  - [ ] You know that **new logic for this course** will normally grow under **`src/`** (additional files and modules later), not by editing artifacts under **`target/`**.
- **Evidence:** A short note (a few sentences or bullets is enough) or annotations you keep for yourself—your choice. If you like learning in the editor, a one-line comment above `fn main()` marking it as the process entry is fine; keep the program compiling with `cargo run`.

**After this lesson:** When you are curious about **release** builds, run `cargo build --release` from the crate root and compare **`target/release/`** to **`target/debug/`** (binary size and path). That exploration is optional here and does not change the lesson contract.
