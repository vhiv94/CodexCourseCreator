# Print structured output with `println!`

**Cited contract:** [`course/spine.yaml`](../../spine.yaml) (M1 → Ch1 → L3), [`course/overview.md`](../../overview.md) (Log Summary CLI arc). **Builds on:** [L2 — Read the crate layout and `main`](L2.md).

Rust’s usual first stop for **printing a line to the terminal** is the **`println!` macro**. It takes a **format string** and, when you need them, **extra arguments** that are spliced into placeholders. At compile time the macro expands into code that handles formatting and writes to **standard output**, then appends a **newline**—so each `println!` call is one line of output unless you embed explicit `\n` inside the string (usually you reach for extra `println!` calls when you want a readable “banner”).

You will see the `!` on many Rust names: **`println!` is a macro**, not a plain function. You do not need the full macro story yet; treat the `!` as a signal that the construct gets **expanded at compile time** and can accept a flexible argument shape that ordinary functions cannot.

For this project step, the goal is a **small welcome screen**: enough structure that a new user immediately knows **which tool** this is and **what it is for** (reading log lines and building summaries over time), not a final product UI.

## Assigned task(s)

- **Build/Change:** In **`src/main.rs`**, replace the default **Hello, world!**-style output with a **short welcome** for the Log Summary CLI. Use **`println!` at least once with interpolated data**—for example the tool name from a **`let`** binding, a version string, or a crate constant—so you are not only printing fixed literals.
- **Expected outcomes / acceptance criteria:**
  - [ ] Output **names the tool** in a way that matches your project intent (for example “Log Summary” or your crate’s display name).
  - [ ] Output states or clearly implies the **purpose** (working with log lines toward summaries), even in one plain sentence.
  - [ ] At least one line uses **`println!` formatting** with a value plugged into the format string (not only static text with no placeholders).
- **Evidence:** `cargo run` shows your welcome; the program still compiles cleanly.

**Pacing note:** A few `println!` lines are enough. Save fancy borders, colors, and argument parsing for later lessons.
