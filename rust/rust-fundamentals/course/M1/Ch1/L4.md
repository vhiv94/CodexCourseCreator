# Capture one line of user input

**Cited contract:** [`course/spine.yaml`](../../spine.yaml) (M1 → Ch1 → L4), [`course/overview.md`](../../overview.md) (Log Summary CLI arc). **Builds on:** [L3 — Print structured output with `println!`](L3.md).

A CLI becomes *interactive* the moment it can take input and respond. In Rust, the most common “first” path is:

- **store text in a `String`** (owned, growable text)
- **use `std::io`** to read from **standard input** (stdin) into that string

This lesson is intentionally small: you will ask the user for **one sample log line**, read **one line** of text from stdin, then print it back out so you can confirm you captured it correctly. Later chapters will turn that same text into parsed fields and real summaries.

One detail to keep in the back of your head: when you read a line from stdin, the input typically includes a trailing newline (`\n`). You do not need to perfectly sanitize input yet, but you should notice whether your echo includes an extra blank line and decide whether to print the raw line or a trimmed version.

## Assigned task(s)

- **Build/Change:** In `src/main.rs`, extend your Log Summary CLI so it:
  - prints a prompt asking for **one sample log line**
  - reads **one line** from **stdin** into a **mutable `String`**
  - prints a confirmation that includes the text you captured (an “echo” is fine)
- **Expected outcomes / acceptance criteria:**
  - [ ] You bring `std::io` into scope and use it intentionally (for example by calling into stdin).
  - [ ] You store interactive input in a **mutable `String`**.
  - [ ] `cargo run` lets you type a line and see the program reuse that text in output.
- **Evidence:** Run the program, type a realistic-looking log line (anything is fine), and capture the output showing the echo/confirmation.

**Optional — piping refresher:** Try driving stdin without typing by hand (one example is piping a single line into `cargo run`). The goal is just to notice that stdin can come from *either* a person *or* another program—this will matter once you start reading real log files.
