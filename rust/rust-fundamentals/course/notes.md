# Cargo

- handles package management
- building projects
    - `cargo new <project-name>` creates a new project 
        - by default `--bin` flag to create a simple binary project for clis
        - add the `--lib` flag to create a library that can be easily linked with another crate
        - sounds a lot easier than `.dll`s
    - `cargo run` compiles, links, and runs the resulting executable
    - `cargo build` just compiles and links
        - by default `--debug` is enabled and outputs debuginfo files, but skips the optimizer
        - add the `--release` flag to automatically enable the optimizer, but skip debuginfo files
        - cargo build gracefully manages the build process that would have to be written itself with C
